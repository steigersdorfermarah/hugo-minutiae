Unser Projekt

Minutiae ist eine Open Source, MIT Lizenzierte Software für ein Fingerabdrucksystem und ein Kooperationsprojekt mit dem OTELO Gmunden. Während Minutiae das Frontend, das Backend und die Datenbank bereitstellt, übernimmt das Co-Projekt die Hardware und den hardwarenahen Code. Die Projektidee ist ein System, in dem Benutzer und Administratoren existieren. Benutzer können Fingerabdrucksensoren verwenden und Türen entsprechend ihrer Berechtigungen ent- / verriegeln. Für Administratoren gibt es eine Weboberfläche, über die sie verschiedene Systemeinstellungen vornehmen können. Dies zB. beinhaltet das Hinzufügen neuer Benutzer und Türen.

Hardware

Was wird benötigt?

  • Ein Raspberry Pi (= Mikrocontroller) + benutzerdefiniertes Shield pro Tür
  • Ein Fingerabdrucksensor pro Tür
  • Ein funktionierendes lokales Netzwerk (=> Router/Switch/..), aber nicht unbedingt eine Internetverbindung für das laufende System

    • Jeder Mikrocontroller verwaltet eine einzelne Tür und hat auch ein eigenes Backend, Frontend und darauf laufende Datenbank. Der Vorteil dieses dezentralen Systems besteht darin, dass selbst wenn eine Tür aus irgendeinem Grund von den anderen Türen getrennt ist, es funktioniert immer noch von selbst (auch wenn Änderungen dafür natürlich nicht mehr wirksam sind), und es gibt auch keinen zentralen Server, wenn das Sterben das ganze System funktionsunfähig machen würde.

    Frontend

    Das Frontend ist in React geschrieben und bietet zusammengefasst folgende Funktionalität:

  • Hinzufügen neuer Benutzer und Admins und speichern von weiteren Informationen wie Adresse, vollständiger Name, Telefonnummer usw., falls gewünscht.
  • Hinzufügen eines Fingerabdruckes für einen vorhandenen Benutzer.
  • Zugang für bestimmte Türen gewären.
  • Hinzufügen neuer Türen und laden der Konfigurationsdateien für ihren Controller.
  • Logs der Systemnutzung anzeigen

    • Backend

    Das Backend von NestJ stellt einerseits die Daten für das Frontend über die REST-Api bereit und übernimmt über MQTT auch die Kommunikation mit der Fingerabdrucksensor-Steuerungssoftware. Dies wird zB benötigt zum Beibehalten der Protokolle, zum Abrufen der Fingerabdrücke für einen neuen Benutzer und zum Überprüfen, ob der Benutzer Zugang für eine Tür hat, wenn er versucht, einzutreten.

    Database

    Als Datenbank-Technologie kommt MariaDb mit Galera Cluster zum Einsatz. Da das System dezentralisiert ist, sorgt der Galera Cluster für die Synchronisierung aller Datenbanken. Damit egal an welchem Raspberry-Frontend der Admin Änderungen am System vornimmt, es an jeder Tür wirksam wird.
